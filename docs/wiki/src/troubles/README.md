# 🛠️ 트러블슈팅 및 기술 최적화 기록

> **문서 목적**: 프로젝트 개발 및 운영 과정에서 마주한 기술적 난제들과 이를 해결하기 위한 최적화 과정을 기록합니다. 비슷한 문제가 발생했을 때 빠른 해결을 위한 가이드로 활용합니다.

---

## 📂 카테고리별 목록

### 1. 백엔드 및 서비스 로직
개발 초기 모놀리식 구조에서 발생한 성능 저하와 구조적 결함을 해결한 과정입니다.
- **[자소서 생성 로직 최적화](./chat_logic_optimize.md)**: 1,000줄 이상의 거대 파일 분리, 루프 내 중복 검색 제거, JSONParser 도입 및 비동기 폴링(Async Polling) 구현 사례.

### 2. AI 인프라 및 모델 서빙 (RunPod)
GPU 서버리스 환경에서 EXAONE 3.5 모델을 안정적으로 서빙하기 위한 트러블슈팅 기록입니다.
- **[RunPod Serverless 가이드](./runpod/serverless.md)**: Flash 패키징 오류, 401 인증 문제, CUDA OOM 대응 및 의존성 관리 전략.
- **[vLLM 추론 최적화](./runpod/vllm.md)**: vLLM 엔진을 이용한 고속 추론 설정 및 메모리 관리 이슈.

## 💡 기록 원칙
- **증상(Symptom)**: 어떤 에러 메시지나 현상이 발생했는지 명확히 기록합니다.
- **원인(Root Cause)**: 로그 분석이나 코드 추적을 통해 밝혀낸 기술적 원인을 기술합니다.
- **해결(Resolution)**: 적용한 코드 수정이나 설정 변경 사항을 공유합니다.
- **예방(Prevention)**: 동일 이슈 재발 방지를 위한 체크리스트를 추가합니다.
