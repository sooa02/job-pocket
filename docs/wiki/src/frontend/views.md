# 🖥️ Job-Pocket 뷰(Views) 상세

> **문서 목적**: 프론트엔드 각 뷰의 상세 로직과 컴포넌트 인터랙션을 기술한다.  
> **최종 수정일**: 2026-04-26  
> **버전**: v0.3.0 (분기 로직 및 퀵 액션 반영)

---

## 1. Chat View (`chat_view.py`)

채팅 인터페이스와 자소서 생성 파이프라인의 핵심 제어부입니다.

### 1.1 생성 및 수정 분기 로직 (`is_revision_request`)
사용자의 입력 메시지에 '수정', '다듬어', '줄여' 등의 키워드가 포함되어 있고 직전 대화 결과물이 존재하는 경우, 신규 생성이 아닌 **수정 모드**로 동작합니다.

### 1.2 진행 상태 시각화 (`generate_response_with_progress`)
파이프라인 실행 중 사용자에게 총 4단계의 진행 상황을 표시합니다.
1. **분석**: 입력 내용 구조화 (Step 1)
2. **생성/수정**: EXAONE 초안 작성 또는 기존 결과 수정 (Step 2)
3. **정제**: 문장 흐름 및 표현 정리 (Step 3)
4. **최종 점검**: 글자 수 조정 및 평가 조립 (Step 4~5)

### 1.3 보완 포인트 및 퀵 액션
AI의 평가 결과 중 '보완 포인트' 섹션은 개별 버튼으로 렌더링됩니다.
- **적용 버튼**: 클릭 시 해당 포인트를 해결하기 위한 최적의 프롬프트(예: "첫 문장을 더 선명하게 수정해줘")를 생성하여 `st.session_state.pending_prompt`에 할당하고 즉시 실행합니다.
- **피드백(👍/👎)**: 결과물에 대한 만족도를 수집하여 UI에 반영합니다.

## 2. Resume View (`resume_view.py`)

사용자의 개인 스펙 정보를 관리하는 설정 화면입니다.

### 2.1 탭(Tabs) 구조
정보의 양이 방대하므로 `st.tabs`를 이용해 세 섹션으로 구분했습니다.
- **👤 인적사항**: 성명, 성별 등 기본 정보.
- **🎓 학력**: 학교, 전공 등 학업 정보.
- **🏆 경력 및 스펙**: 인턴 경험, 수상 내역, 기술 스택.

### 2.2 JSON 기반 저장 전략
프론트엔드에서 취합된 모든 폼 데이터는 하나의 딕셔너리로 묶여 백엔드로 전송됩니다. 백엔드는 이를 JSON 문자열로 저장하므로, 프론트엔드에서 필드를 추가하더라도 DB 스키마 변경 없이 유연한 확장이 가능합니다.

## 3. Auth View (`auth_view.py`)

### 3.1 폼 제출 최적화
`st.form`과 `st.form_submit_button`을 사용하여 엔터 키 입력 시 자연스러운 제출을 지원합니다. 로그인 성공 시 사용자 정보(`user_info`)를 세션에 저장하고 자동으로 채팅 화면으로 전환됩니다.

---

## 4. 핵심 UI 컴포넌트 (`utils/ui_components.py`)

### 4.1 커스텀 CSS 주입
Streamlit의 기본 디자인 한계를 극복하기 위해 `apply_custom_css` 함수를 통해 다음 스타일을 적용합니다.
- **채팅 말풍선 최적화**: 가독성 높은 배경색과 간격 적용.
- **사이드바 하단 고정**: 로그아웃 버튼을 사이드바 하단에 배치.
- **진행 카드 스타일**: `progress-card` 클래스를 정의하여 단계별 안내 상자 디자인.

---

## 5. 데이터 흐름 요약

1. **Input**: 사용자 입력 (채팅창 또는 퀵 액션 버튼).
2. **Step 1 (Parse)**: `api_client.parse_request_api` 호출 → `parsed_data` 획득.
3. **Logic Branch**: `is_revision_request`에 따라 `step-draft` 또는 `step-revise` 호출.
4. **Refine/Fit**: 순차적 API 호출로 결과물 고도화.
5. **Final**: `build_final_response_api`로 최종 텍스트 수신 및 세션 기록.
